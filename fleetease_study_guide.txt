Fleetease Project - Interview Study GuideThis guide provides a comprehensive overview of the Fleetease project to help you prepare for your interview. It covers the architecture, technology stack, database schema, and key functionalities.1. Project OverviewFleetease is a comprehensive Fleet Management and Booking System designed for both B2C (Business-to-Consumer) and B2B (Business-to-Business) use cases. It allows customers to book vehicles for routes, while administrators and company admins can track vehicles in real-time, manage drivers, and handle invoicing.2. Technology StackThe project is built using a modern JavaScript/Node.js stack (PERN stack variation).Backend:Runtime: Node.jsFramework: Express.jsDatabase: PostgreSQL (using the `pg` driver)Real-time Communication: Socket.IO (used for live GPS tracking and potentially the chatbot)Authentication: JWT (JSON Web Tokens) & `bcrypt` for password hashingDeployment/Infra: Docker & Docker Compose (`docker-compose.yml` sets up a postgres DB and the Node app)Frontend:Framework: React.js (Bootstrapped with Create React App / `react-scripts`)Routing: React Router v6Styling: Bootstrap 5.3 & custom CSS (`App.css`)Maps: Leaflet & React-Leaflet (for the real-time Live Map feature)State/Requests: Axios for API calls, React Hooks.Real-time: `socket.io-client`3. Database Schema (PostgreSQL)The database structure is well-normalized and handles various entities required for fleet management:`users`: Central auth table. Roles include `admin`, `customer`, `driver`, `company_admin`. Contains loyalty points and hashed passwords.`vehicles`: Stores fleet data (number, type, capacity, assigned driver, availability status).`drivers`: Extended profile for driver users (license number, status, joined date). Linked to `users` via `user_id`.`routes`: Pre-defined routes with source, destination, distance (km), and fare per km.`bookings`: The core transactional table. Links customer, driver, vehicle, and route. Handles B2C/B2B context, pickup/drop locations, fares, and approval statuses.`payments` & `invoices`: Financial tracking. `invoices` are heavily used for B2B company billing.`vehicle_locations`: Telemetry data (latitude, longitude, speed, fuel level). Updated via GPS endpoints.`maintenance`: Service logs and costs for vehicles.`gallery`: Publicly visible images/achievements managed by admins.4. Key Features & FunctionalitiesAuthentication & AuthorizationRole-based access control. The React frontend uses a `<ProtectedRoute>` component that restricts pages (e.g., `/admin`, `/drivers`) based on user roles (`admin`, `company_admin`, `driver`, `customer`).JWT tokens are passed in headers for API requests.Booking SystemSupports B2C (direct user bookings) and B2B (corporate bookings).Bookings go through an approval workflow (e.g., pending -> approved/active -> completed).Real-Time GPS TrackingA `/map` route in the frontend displays a live map using `react-leaflet`.The backend emits location updates via Socket.IO (`io.on('join_vehicle')`) which the frontend listens to, animating vehicle markers on the map.Admin & Company DashboardsAdmin Summary: High-level metrics, revenue, active bookings.Fleet Management: Admins can view/add/edit vehicles (`/vehicles`) and drivers (`/drivers`).Invoicing: B2B bookings generate invoices which can be managed and tracked for payment status (`/invoices`).Interactive ChatbotA rule-based smart chatbot is integrated into the frontend (`Chatbot.jsx`) to assist users with contextual B2B/B2C queries.5. Potential Interview Talking PointsIf asked about this project in an interview, you should highlight:Handling Multiple Roles: Explain how you managed different user roles (Admin vs. Customer vs. Driver) both in the backend (middleware) and frontend (React Router protected routes).Real-time Features: Discuss the integration of Socket.IO with React-Leaflet to provide live vehicle tracking. This demonstrates knowledge of WebSockets over standard HTTP REST.Database Design: Mention how the schema connects users to roles (e.g., the `users` table linked to a specific `drivers` table) and how bookings act as the central pivot table linking routes, vehicles, drivers, and customers.B2B vs B2C Logic: Highlight the business logic complexity, such as handling standard payments for B2C versus generating invoices and approval workflows for B2B clients.Good luck!
